# Лабораторна робота — Варіант 1

- Демонстрація механізму парування класів і створення нащадків через `Matchmaker`.
- Використовуються атрибути `CoupleAttribute` для опису сумісності та параметрів створення дитини.
- Класи, задіяні в демо: `Human`, `HumanMale`, `Girl`, `PrettyGirl`, `Matchmaker`, `CoupleAttribute`.

Як працює демо

- Програма вибирає пару об'єктів (чоловік + жінка) і оцінює взаємну симпатію на основі атрибутів.
- Якщо обидва симпатизують одне одному і знайдено опис дитини у `CoupleAttribute`, створюється об'єкт-нащадок заданого типу.
- Ім'я дитини може генеруватися методом у батьківському класі або братися з доступного `Name`.
- При невдалому паруванні виводиться коротке повідомлення — без деталей помилок.

Запуск 

```powershell
dotnet build LabVariant1.sln
dotnet run --project Variant1.csproj
```

Демо відключено у неділю (програма перевіряє день тижня і виводить повідомлення, якщо сьогодні неділя).

Де шукати код

- Основна логіка демо: каталог `src`, файли `Program.cs` та `Matchmaker.cs`.
- Опис атрибутів: `src/Attributes/CoupleAttribute.cs`.

Збирання та запуск

Для збірки та запуску з кореневого каталогу проєкту виконайте:

```powershell
dotnet build LabVariant1.sln
dotnet run --project Variant1.csproj
```

Під час виконання програма виводить короткі повідомлення в консоль і запитує деякі значення (наприклад, ім'я файлу для збереження/завантаження або розміри масиву). Далі слідуйте підказкам у консолі.

Файли

- Головний код знаходиться в каталозі `src`.
- Збірні артефакти з'являються в `bin/Debug/net*` після збірки.


# Лабораторна робота №4 — Варіант 1 (Колекції і продуктивність)

Цей репозиторій розширено для лабораторної роботи №4. Мета: порівняти продуктивність стандартних змінних колекцій (`List`, `Dictionary`), незмінних колекцій (`System.Collections.Immutable`) і відсортованих структур (`SortedList`, `SortedDictionary`) при типових операціях: побудова/додавання, пошук за ключем, пошук за значенням і операція `Contains`.

Що додано для Лабораторної роботи №4:

- `src/Collections/TestCollectionsLab4.cs` — клас, що генерує набори даних і виконує серію вимірювань, усереднюючи результати по кількох ітераціях.
- Оновлено `src/Program.cs` для виклику бенчмарків Lab4 після попередніх демонстрацій.
- В `Variant1.csproj` додано посилання на пакет `System.Collections.Immutable`.

Як запускати бенчмарки:

1. `dotnet build`
2. `dotnet run`

Після запуску програма виведе результати вимірювань у консоль. Можна змінити розмір набору при створенні `TestCollectionsLab4` в коді.

Коротка інтерпретація очікуваних результатів:

- `List.Contains` — O(n) по середньому часу. Пошук у стандартному списку повільніший для великих наборів.
- `Dictionary.ContainsKey` — звичайно O(1) (хеш-пошук); `ContainsValue` — O(n).
- Незмінні колекції можуть мати вищі накладні витрати при побудові, але вигідні для паралельного/неблокуючого доступу.
- Відсортовані колекції підтримують логарифмічний або близький до логарифмічного пошук за ключем і гарантують впорядкованість.

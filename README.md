# Лабораторна робота — Variant1

Цей репозиторій містить навчальну програму на C# та приклади коду.

## Мета

Пояснити та закріпити базові концепції об'єктно-орієнтованого програмування (ООП) на прикладі простого проєкту: створення класів `Person`, `Student`, роботи з екзаменами (`Exam`) та запуску програми (`Program.cs`).

## Структура файлів

- `Person.cs`: базовий клас для опису людини (поля, властивості, конструктор).
- `Student.cs`: клас-спадкоємець `Person`, додає студентські властивості та методи.
- `Exam.cs`: логіка, пов'язана з іспитами (оцінки, результати).
- `Program.cs`: точка входу — приклади створення об'єктів і викликів методів.
- `Variant1.csproj`: конфігурація проєкту .NET.

## Що ми вивчили — теорія (детально)

1) Об'єктно-орієнтоване програмування (ООП)

- Основна ідея: моделювання системи у вигляді об'єктів, які мають стан (поля/властивості) і поведінку (методи).
- Переваги: інкапсуляція, повторне використання коду, зрозуміліша модель предметної області.

2) Класи та об'єкти

- Клас — шаблон або тип, що описує спільні властивості та поведінку.
- Об'єкт (екземпляр класу) — конкретна реалізація класу з власними значеннями полів.
- Приклад:

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }

    public void Greet() => Console.WriteLine($"Hello, I'm {Name}.");
}
```

3) Інкапсуляція

- Приховування внутрішнього стану класу та надання контролю доступу через властивості (`get`/`set`) або методи.
- Захищає від некоректного використання та дозволяє змінювати внутрішню реалізацію без впливу на зовнішній код.

4) Наслідування

- Механізм, коли клас (підклас) успадковує поля й методи базового класу.
- Приклад: `Student : Person` — студент має все, що є в `Person`, та додає свої поля (наприклад, `StudentId`, `Grades`).

5) Поліморфізм

- Дозволяє трактувати об'єкт підкласу як об'єкт базового типу і викликати перевизначені (overridden) методи під час виконання.
- Реалізується через віртуальні методи (`virtual`) та їх перевизначення (`override`).

6) Абстракція і інтерфейси

- Абстрактний клас може містити не реалізовані методи — загальний контракт для підкласів.
- Інтерфейси визначають набір методів і властивостей, які клас зобов'язаний реалізувати.

7) Конструктори та ініціалізація

- Конструктор — метод для ініціалізації об'єкта при створенні. Можуть бути кілька конструкторів (перевантаження).

8) Колекції та LINQ (огляд)

- Вбудовані колекції: `List<T>`, `Dictionary<TKey,TValue>`, `Array` тощо.
- `List<T>` — найпоширеніша динамічна колекція для зберігання об'єктів одного типу.
- LINQ (Language Integrated Query) дозволяє фільтрувати, проєктувати та агрегувати дані в колекціях декларативно.

Приклад роботи зі `List<Exam>`:

```csharp
var exams = new List<Exam> { /* ... */ };
var passed = exams.Where(e => e.Score >= 60).ToList();
```

9) Обробка винятків

- Використовуємо `try { } catch { } finally { }` для перехоплення помилок виконання і забезпечення стабільності програми.

10) Ввід/вивід файлів

- `File.ReadAllText`, `File.WriteAllText`, `StreamReader`/`StreamWriter` — стандартні засоби для роботи з файлами.

11) Статичні члени

- `static` поля або методи належать типу, а не окремому об'єкту. Використовуються для зберігання загального стану або допоміжних функцій.

12) Рекомендовані практики

- Підтримуйте чисті й короткі методи.
- Використовуйте приватні поля з публічними властивостями.
- Пишіть тести для критичної логіки (якщо можливо).

## Як запустити проект

1. Відкрийте термінал у папці з проєктом.
2. Збудуйте проект:

```powershell
dotnet build
```

3. Запустіть:

```powershell
dotnet run
```

(Команди працюють, якщо на машині встановлений .NET SDK.)
